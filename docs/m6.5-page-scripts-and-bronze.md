# M6.5 — Page scripts reorganization + bronze layer

**Status:** pending (retrofit milestone — augments M4 recorder, M5 storage, M6 executor without redoing them)

**Source:** [medallion-and-workers.md](medallion-and-workers.md) §1.1 + §2

## Goal

Two retrofit changes that share a milestone because they both touch the recorder + executor write paths:

1. **JS subdirectory split.** Move the single [src/rpa_recorder/assets/recorder_inject.js](../src/rpa_recorder/assets/recorder_inject.js) (M4) into a typed `page_scripts/` package with `recorder/`, `replay/`, `shared/` subdirs and stub files for future scripts (`dom_snapshot.js`, `a11y_tree.js`, `locator_healing.js`, `modal_detector.js`).
2. **Bronze layer foundation.** A `BronzeStore` Protocol with a `LocalFilesystemStore` v1, a `bronze_artifacts` pointer table, and recorder/executor write paths that route raw envelopes, screenshots, DOM dumps, accessibility snapshots, HAR files, and trace bundles through the store.

These are foundational for M9 (LLM call bronze writes), M10 (recovery selectors → bronze), M11.5 (silver promotion + Parquet compaction), and M12 (medallion-aware FastAPI endpoints).

## Public API

### `page_scripts/__init__.py`

```python
def load(script: str) -> str:
    """Load a JS file by relative path. `'recorder/inject'` -> `recorder/inject.js`.

    Raises FileNotFoundError for unknown paths.
    """

def bundle(*scripts: str) -> str:
    """Concatenate scripts in order, separated by `\\n;\\n`.

    The leading semicolon prevents JS ASI bugs when one script ends in an
    expression and the next starts with `(`, `[`, or `/`.
    """
```

### `medallion/bronze_store.py`

```python
from typing import Protocol, BinaryIO

class BronzeStore(Protocol):
    async def put(self, path: str, data: bytes | BinaryIO) -> str: ...
    async def append_line(self, path: str, line: str) -> None: ...
    async def get(self, path: str) -> bytes: ...
    async def list(self, prefix: str) -> list[str]: ...
    async def delete(self, path: str) -> None: ...
    async def stat(self, path: str) -> tuple[int, str]: ...

class LocalFilesystemStore:
    def __init__(self, root: Path) -> None: ...
    # implements BronzeStore against the local filesystem under `root`
```

## Behavior

### Page-script loader

- `load("recorder/inject")` resolves to package `rpa_recorder.page_scripts.recorder` and reads `inject.js`. Uses `importlib.resources.files(pkg).joinpath(leaf).read_text(encoding="utf-8")`.
- `bundle("a", "b", "c")` returns `f"{a}\n;\n{b}\n;\n{c}"`. The semicolon prevents ASI bugs.
- Both helpers are sync — JS files ship inside the wheel, no I/O contention.

### `LocalFilesystemStore`

- Constructed with an absolute `root: Path`. Methods accept *store-relative* paths only (`"recordings/<id>/raw_events.jsonl"`); the implementation joins under `root` with `pathlib.Path` operations only — no string concat (ruff PTH).
- `put(path, data)` writes atomically: writes to `path + ".tmp"`, then `Path.replace(target)`. Returns `hashlib.sha256(data).hexdigest()`.
- `append_line(path, line)` opens with `"a"` mode (creates parent dirs idempotently via `Path.mkdir(parents=True, exist_ok=True)`), writes `line` followed by `\n`. The file is closed between calls — bronze is the durable record, not a hot-path.
- `delete(path)` is idempotent — `missing_ok=True`.
- `stat(path)` reads the file, returns `(size, sha256)`. We don't memoize because most consumers only call `stat` once per artifact (at registration time).
- All operations use `aiofiles` for true async I/O. (Add `aiofiles` to `pyproject.toml` if not already present.)

### `BronzeWriter`

- `append_event(recording_id, envelope)` is called from `Recorder._on_capture` *before* normalization. Writes one JSON line via `store.append_line(paths.recording_events_jsonl(recording_id), json.dumps(envelope))`. On the *first* envelope for a recording, also creates a `bronze_artifacts` row with `kind="event_jsonl"`, `sha256=""`, `size_bytes=0`. The hash + size are filled in at `finalize_recording` (M11.5 compaction recomputes them when it converts to Parquet).
- `finalize_recording(recording_id, har_bytes, trace_bytes)` is called by `BrowserSession.__aexit__`. Writes HAR and trace bundles via `store.put(...)` and registers their pointer rows. Updates the `event_jsonl` row's sha256/size by `stat()`-ing the file.
- `write_attempt_artifact(run_id, attempt_id, kind, data)` is called by `Executor._capture_failure`. Writes the file via `store.put(...)`, registers a `bronze_artifacts` row with FK on `attempt_id`. Returns the store-relative path so the executor can populate `ExecutionAttempt.screenshot_path` etc.
- `write_llm_call(call_id, payload)` is called by M9 `LLMClassifier` and M10 `LLMReselectStrategy`. Writes the full prompt+response JSON, registers pointer.
- All bronze writes are best-effort: failures are logged via `structlog.error(...)` but never raised. Recording must never fail because bronze write failed.

### Recorder migration

- Loader call site (currently [src/rpa_recorder/browser/recorder.py:40](../src/rpa_recorder/browser/recorder.py:40), `_load_inject_script()`) is replaced with:
  ```python
  from rpa_recorder.page_scripts import bundle
  script = bundle("shared/text_utils", "shared/selectors", "recorder/inject")
  ```
- `Recorder._on_capture` ([src/rpa_recorder/browser/recorder.py:142](../src/rpa_recorder/browser/recorder.py:142)) gains a single `await self._bronze.append_event(self._recording_id, raw)` *before* validation. The bronze write happens regardless of whether validation later fails — bronze is the immutable raw record.
- The async ordering invariant from the M4 docstring (no `await` between `self._sequence += 1` and `self._actions.append(...)`) still holds: the bronze write happens *before* the sequence increment.
- `Recorder.__init__` gains a `bronze: BronzeWriter` parameter (passed through from the CLI / API caller).

### Executor migration

- `Executor._capture_failure` ([src/rpa_recorder/browser/executor.py](../src/rpa_recorder/browser/executor.py)) currently writes:
  - screenshot to `screenshots_dir/<run_id>/<seq>_<attempt>.png`
  - DOM to `dom_dir/<run_id>/<seq>_<attempt>.html`
  - a11y to `dom_dir/<run_id>/<seq>_<attempt>.a11y.json`
- Replace those direct writes with three `BronzeWriter.write_attempt_artifact(...)` calls. The returned store-relative path populates `ExecutionAttempt.screenshot_path` / `dom_snapshot_path` / `accessibility_snapshot_path`.
- The legacy `screenshots_dir` and `dom_dir` constructor params stay (for backward-compat with M6 tests) but become unused once `bronze: BronzeWriter` is provided. Document the deprecation in the docstring.

### `BrowserSession.__aexit__` HAR + trace handoff

- Currently the M3 `BrowserSession` writes HAR + trace to `traces_dir`. Augment `__aexit__` to read those bytes back and call `BronzeWriter.finalize_recording(...)` so they live in bronze too. Eventually the local `traces_dir` write can be removed in a follow-up; for now both writes coexist.

### JS shared utilities

- `shared/text_utils.js` exposes `window.__rpa.shared.text` with `trim`, `accessibleName` lifted from existing inject.js.
- `shared/selectors.js` exposes `window.__rpa.shared.selectors` with `inferRole`, `uniqueCss`, `xpathOf`, `safeRect`, `isVisible`, `nearbyLabels`, `attrs`.
- `recorder/inject.js` keeps `targetSnapshot`, `envelope`, `send`, and event listeners; calls into shared utilities via `window.__rpa.shared.*`.
- All scripts remain IIFEs guarded by `window.__rpa<Name>Loaded` to be idempotent across `add_init_script` + `evaluate(script)` injection.

## Known pitfalls

- **Async context inside event listeners.** `Recorder._on_capture` is already `async`; calling `await self._bronze.append_event(...)` inside it is fine, but don't add an `await` between `self._sequence += 1` and `self._actions.append(...)` — that breaks the ordering invariant noted in the M4 docstring. The bronze write goes *before* the increment.
- **Init-script eval order.** `add_init_script` runs the bundle on every navigation; `evaluate(script)` runs it once for the current document. Each script's IIFE guard (e.g., `window.__rpaSharedTextLoaded`) prevents double-define. Don't strip them.
- **PEP 758 except syntax.** Python 3.14 (PEP 758) lets you write `except A, B:` without parens — see [src/rpa_recorder/browser/recorder.py:147](../src/rpa_recorder/browser/recorder.py:147) for an existing real-world example. It's legal and ruff doesn't complain. Style choice; either works.
- **`runtime_checkable` Protocol vs ruff TC002.** `BronzeStore` is a `typing.Protocol`. Don't add `@runtime_checkable` — we don't need `isinstance(x, BronzeStore)` anywhere, and runtime_checkable Protocols don't compose well with mypy strict. Type-only references can stay under `if TYPE_CHECKING:`. `LocalFilesystemStore` itself is runtime-imported.
- **HAR + trace bytes can be hundreds of MB.** `finalize_recording` should `await` the write outside any DB transaction (open the bronze write, commit the pointer row separately).
- **`assets/` package staying empty.** After deleting `recorder_inject.js`, the `assets/` directory will be empty. Delete the directory entirely — leaving an empty package confuses readers and offers no future use.
- **Backward-compat with M6 executor tests.** The existing M6 tests pass `screenshots_dir` and `dom_dir` directly. Keep accepting them in `Executor.__init__` (deprecated) so the M6 test suite passes unchanged. New tests use `bronze: BronzeWriter`.
